Self-Deployment Guide

How this static dashboard page is deployed and served.

1

Clone Repository

Get the latest version of the dashboard page from GitHub. This page (eek.html) lives in the lab-dashboard repository.

# Navigate to your web server's root or a designated projects folder
cd /var/www
git clone https://github.com/uprootiny/lab-dashboard.git lab.dissemblage.art
# This clones and names the directory for our domain
2

Set Web Root

Ensure your web server configuration points to the directory containing eek.html. We cloned it directly into /var/www/lab.dissemblage.art in the previous step.

# Nginx 'root' directive will point here:
/var/www/lab.dissemblage.art

No extra linking is needed if you clone to the intended web root.

3

Configure Nginx

Set up an Nginx server block to serve the static file. Create or edit a file in /etc/nginx/sites-available/.

# /etc/nginx/sites-available/lab.dissemblage.art.conf
server {
    listen 80;
    server_name lab.dissemblage.art www.lab.dissemblage.art;

    root /var/www/lab.dissemblage.art;
    index eek.html; # Our dashboard file

    location / {
        try_files $uri $uri/ =404;
    }

    # For Let's Encrypt challenges (adjust root if needed)
    location ~ /.well-known/acme-challenge/ {
        allow all;
        root /var/www/html;
    }
}

# Enable the site and test configuration
sudo ln -s /etc/nginx/sites-available/lab.dissemblage.art.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
4

Point DNS & Secure (SSL)

First, ensure your domain's DNS records point to your server's IP. For dissemblage.art, this is managed at Porkbun. Create 'A' records for lab (and optionally www.lab) pointing to your server.

Then, use Certbot with Let's Encrypt to get a free SSL certificate. This requires DNS to be set up correctly.

# Ensure Certbot and its Nginx plugin are installed
sudo apt update && sudo apt install certbot python3-certbot-nginx -y

# Run Certbot - it will find your domain in Nginx and set up HTTPS
sudo certbot --nginx -d lab.dissemblage.art -d www.lab.dissemblage.art

# Certbot automatically sets up renewal. Test it:
sudo certbot renew --dry-run
5

Automate Updates (Self-Deployment)

For a truly self-deploying system, automate pulling changes from GitHub. This page is static, so a simple git pull is enough. This could be triggered by a GitHub Action (pushing to the server), a webhook, or a simple cron job on the server.

# Example: Simple update script (e.g., /usr/local/bin/update-dashboard.sh)
#!/bin/bash
cd /var/www/lab.dissemblage.art || exit 1
git fetch origin
git reset --hard origin/main # Force update to match remote
echo "Dashboard updated at $(date)" >> /var/log/dashboard_updates.log

# Make it executable: chmod +x /usr/local/bin/update-dashboard.sh
# Run via cron (e.g., every 5 mins): */5 * * * * /usr/local/bin/update-dashboard.sh

More advanced setups could use GitHub Actions to SSH into the server and run the pull, or use webhooks to trigger the script, providing more immediate updates.
